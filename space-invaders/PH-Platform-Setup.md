# Space Invaders — Personalisation Hub Platform Setup Guide

## Connected Gaming Experience: Mobile Phone → Portrait Digital Signage

---

## Experience Overview

A customer approaches a portrait digital signage display running the Space Invaders PWA. The screen shows an idle game demo and a dynamic QR code. The customer scans the QR code on their phone, which opens the mobile controller in their browser. A WebSocket session is established automatically, and the customer plays Space Invaders on the big screen using their phone as the arcade controller.

```
┌──────────────────────┐                ┌───────────────────────────┐
│  PORTRAIT SIGNAGE    │◀──WebSocket──▶ │   CUSTOMER MOBILE PHONE   │
│  (PWA Player)        │                │   (Mobile Site)           │
│                      │                │                           │
│  ┌────────────────┐  │   ←  input     │   ┌───────────────────┐   │
│  │ Space Invaders │  │                │   │  [◀]  [FIRE]  [▶] │   │
│  │   Game Canvas  │  │   score/state →│   │                   │   │

│  [QR Code — Idle]    │   scan QR ─────│▶  controller opens        │
```

## 4 URLs to Configure in the PH Platform

### 1. Mobile Site URL
**What it is:** The URL loaded on the customer's phone after they scan the QR code. This is the arcade controller interface.

```
https://your-domain.com/space-invaders-mobile.html
```

**URL Parameters injected at runtime by the QR code:**

| Parameter | Description | Example |
|---|---|---|
| `session` | Session ID generated by the display | `A3F7C21B` |
| `ws` | WebSocket server URL | `wss://ws.yourdomain.com/ws` |
| `store` | Store code (from PH platform) | `STORE001` |
| `endedUrl` | URL to redirect to when game ends | `https://your-domain.com/space-invaders-ended.html` |

**Full example (as encoded in QR):**
```
https://your-domain.com/space-invaders-mobile.html?session=A3F7C21B&ws=wss://ws.yourdomain.com/ws&store=STORE001&endedUrl=https://your-domain.com/space-invaders-ended.html
```

---

### 2. PWA Player URL
**What it is:** The URL loaded on the portrait digital signage display. This runs the full Space Invaders game and generates the QR code for the customer to scan.

```
https://your-domain.com/space-invaders-display.html
```

**URL Parameters configured in PH Device Management:**

| Parameter | Description | Example | Source |
|---|---|---|---|
| `ws` | WebSocket server URL | `wss://ws.yourdomain.com/ws` | HQ Admin → Device Management |
| `store` | Store code | `STORE001` | HQ Admin → Store setup |
| `device` | Device UDID | `DISP-LONDON-01` | HQ Admin → Device registration |
| `mobileUrl` | Base URL for the mobile controller | `https://your-domain.com/space-invaders-mobile.html` | HQ Admin → Device Management |
| `endedUrl` | Redirect URL for game end | `https://your-domain.com/space-invaders-ended.html` | HQ Admin → Device Management |

**Full example (entered into PH Device Management):**
```
https://your-domain.com/space-invaders-display.html?ws=wss://ws.yourdomain.com/ws&store=STORE001&device=DISP-LONDON-01&mobileUrl=https://your-domain.com/space-invaders-mobile.html&endedUrl=https://your-domain.com/space-invaders-ended.html
```

---

### 3. Session Creation URL
**What it is:** REST API endpoint used by the PH Platform to programmatically create game sessions. Returns all four URLs needed for a full session, including the encoded display URL and mobile URL.

```
POST https://ws.yourdomain.com/api/sessions/create
```

**Request body (JSON):**
```json
{
  "storeCode": "STORE001",
  "deviceUid": "DISP-LONDON-01"
}
```

**Response (JSON):**
```json
{
  "sessionId":  "A3F7C21B",
  "wsUrl":      "wss://ws.yourdomain.com/ws",
  "displayUrl": "https://your-domain.com/space-invaders-display.html?session=A3F7C21B&ws=wss%3A%2F%2Fws.yourdomain.com%2Fws&store=STORE001&device=DISP-LONDON-01",
  "mobileUrl":  "https://your-domain.com/space-invaders-mobile.html?session=A3F7C21B&ws=wss%3A%2F%2Fws.yourdomain.com%2Fws&store=STORE001",
  "endedUrl":   "https://your-domain.com/space-invaders-ended.html?store=STORE001",
  "created":    "2026-03-12T10:00:00.000Z"
}
```

**Usage:** The PH Platform can call this endpoint when a new display boots, or at the start of each game round, to generate a fresh session with a new QR code.

---

### 4. User Disconnect / Redirect URL
**What it is:** The page the customer's phone is redirected to when the game ends (all lives lost) or if they disconnect mid-game. Shows the final score, a congratulations/retry message, and CTAs to re-engage with the store experience.

```
https://your-domain.com/space-invaders-ended.html
```

**URL Parameters automatically appended at redirect time:**

| Parameter | Description | Example |
|---|---|---|
| `score` | Player's final score | `2450` |
| `reason` | Why the page appeared | `gameover` or `disconnect` |
| `session` | Session ID for reference | `A3F7C21B` |
| `store` | Store code | `STORE001` |

**Optional parameters to configure (set in PH):**

| Parameter | Description | Example |
|---|---|---|
| `playAgain` | URL to restart the game | `https://your-domain.com/space-invaders-mobile.html` |
| `storeUrl` | Link to explore the store (mobile store site) | `https://your-domain.com/store` |
| `staffUrl` | Link to contact staff / join queue | `https://your-domain.com/queue` |

**Full example redirect URL (generated at game end):**
```
https://your-domain.com/space-invaders-ended.html?score=2450&reason=gameover&session=A3F7C21B&store=STORE001
```

## WebSocket Message Protocol

All messages are JSON. The session server routes messages between the display and mobile using the Session ID as the room key.

### Mobile → Display (controller inputs)

| Message | Description |
|---|---|
| `{ type: 'input', action: 'left' }` | Player pressing left — continuous move |
| `{ type: 'input', action: 'right' }` | Player pressing right — continuous move |
| `{ type: 'input', action: 'stop' }` | Player released move button |
| `{ type: 'input', action: 'fire' }` | Player pressed fire |

### Display → Mobile (game state)

| Message | Description |
|---|---|
| `{ type: 'scoreUpdate', score, lives, level }` | Live score/status update |
| `{ type: 'gameStart', sessionId }` | Game has started |
| `{ type: 'gameOver', score }` | Game ended — triggers redirect |

### Session Server → Both

| Message | Description |
|---|---|
| `{ type: 'registered', role, sessionId }` | Confirms registration |
| `{ type: 'player_connected' }` | Sent to display when mobile joins |
| `{ type: 'game_ready' }` | Sent to mobile when display is ready |
| `{ type: 'peer_disconnected', role }` | Other party has disconnected |

## Deployment

### File Structure
```
/                                       ← Web server root
├── space-invaders-display.html         ← PWA Player URL
├── space-invaders-mobile.html          ← Mobile Site URL
├── space-invaders-ended.html           ← Disconnect / Redirect URL
└── (session-server.js runs separately) ← Node.js WebSocket server
```

### Session Server Deployment

```bash
# Install dependencies
npm install ws

# Set environment variables
export PORT=8080
export DISPLAY_BASE="https://your-domain.com/space-invaders-display.html"
export MOBILE_BASE="https://your-domain.com/space-invaders-mobile.html"
export ENDED_BASE="https://your-domain.com/space-invaders-ended.html"

# Start the server
node session-server.js
```

**Recommended:** Run behind nginx with SSL termination so WebSocket connections use `wss://` and REST calls use `https://`.

## Local Demo (No Server Required)

The experience includes a **BroadcastChannel demo mode** that works on a single device (e.g. for internal demos or testing). No session server is needed.

1. Serve the files via any local web server:
   ```bash
   npx serve .
   # or
   python3 -m http.server 8080
   ```
2. Open `space-invaders-display.html` in one browser window — note the Session ID shown
3. Open `space-invaders-mobile.html?session=XXXXXXXX` in another tab (same browser)
4. The display and mobile connect automatically via BroadcastChannel
5. Play Space Invaders — the mobile controls the display

## Technical Architecture

```
PH HQ Admin
 └─ Device Management
     ├─ WebSocket Base URL  → wss://ws.yourdomain.com
     ├─ Auth Token          → (PH API auth)
     ├─ Device UDID         → DISP-LONDON-01
     └─ PWA Player URL      → space-invaders-display.html?ws=...&store=...&device=...
                                          │
                                          ▼
                              Portrait Signage Display
                              (PWA — space-invaders-display.html)
                                          │
                              Generates Session ID (e.g. A3F7C21B)
                              Renders QR code linking to Mobile Site URL + Session ID
                              Connects to WebSocket as 'display' role
                                          │
                              Customer scans QR code
                                          │
                                          ▼
                              Customer Mobile Phone
                              (space-invaders-mobile.html?session=A3F7C21B&ws=...)
                              Connects to WebSocket as 'mobile' role
                                          │
                         ╔════════════════╧════════════════╗
                         ║  Session Server — /ws           ║
                         ║  Routes messages between        ║
                         ║  display ←→ mobile              ║
                         ╚════════════════╤════════════════╝
                                          │
                              Game Starts on Display
                              Mobile sends: input events (left/right/fire)
                              Display sends: score/lives/level updates
                                          │
                              Game Over / Disconnect
                                          │
                                          ▼
                              Mobile redirects to:
                              space-invaders-ended.html?score=2450&reason=gameover
```
